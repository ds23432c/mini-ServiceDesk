# 🚀 mini-ServiceDesk — Инструкция по развертыванию

## Что нужно установить (один раз)

| Инструмент | Версия | Откуда |
|---|---|---|
| Node.js | 18+ | https://nodejs.org (LTS) |
| MySQL Workbench | есть | уже установлен |

---

## Шаг 1 — Создать базу данных

1. Открой **MySQL Workbench**
2. Подключись к локальному серверу (root / root)
3. В верхнем меню: **File → Open SQL Script...**
4. Выбери файл `schema.sql` из папки проекта
5. Нажми кнопку ⚡ **Execute All** (или Ctrl+Shift+Enter)
6. В левом дереве появится база `servicedesk` с таблицами и тестовыми данными

---

## Шаг 2 — Установить зависимости

Открой терминал (cmd / PowerShell / Terminal), перейди в папку проекта:

```bash
cd mini-servicedesk
npm install
```

Это загрузит Express, mysql2, cors (примерно 30 секунд).

---

## Шаг 3 — Запустить сервер

```bash
npm start
```

Появится:

```text
✅  mini-ServiceDesk запущен
   Открой браузер: http://localhost:3000
```

---

## Шаг 4 — Открыть в браузере

Перейди по адресу: **http://localhost:3000**

Готово! Приложение полностью работает.

## Настройка подключения к БД

Теперь проект берёт настройки из переменных окружения. Это нужно для деплоя в интернет.

Пример переменных:

```bash
PORT=3000
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=root
DB_NAME=servicedesk
DB_PORT=3306
```

Если запускаешь локально, эти значения можно не задавать — будут использованы значения по умолчанию.

## Как выложить проект в интернет

### Вариант 1 — самый простой: Render + внешняя MySQL

1. Создай репозиторий на GitHub и залей туда проект.
2. Создай базу MySQL в облаке:
   - Railway
   - Aiven
   - DigitalOcean Managed Database
   - любой другой MySQL-хостинг
3. Импортируй в эту базу файл `schema.sql`.
4. На Render создай новый Web Service из GitHub-репозитория.
5. Укажи команду запуска:
   ```bash
   npm start
   ```
6. В настройках Render добавь переменные окружения:
   - `PORT`
   - `DB_HOST`
   - `DB_USER`
   - `DB_PASSWORD`
   - `DB_NAME`
   - `DB_PORT`
7. После деплоя Render выдаст публичный URL. По нему сможет зайти любой человек.

### Вариант 2 — VPS + Nginx

1. Арендуй VPS.
2. Установи Node.js 18+ и MySQL.
3. Скопируй проект на сервер.
4. Импортируй `schema.sql` в MySQL.
5. Создай на сервере `.env` или задай переменные окружения.
6. Запусти приложение через `npm start` или `pm2`.
7. Поставь Nginx как reverse proxy.
8. Подключи домен и HTTPS через Let’s Encrypt.

## Что важно не забыть

- База данных тоже должна быть доступна из интернета, а не только на твоём ПК.
- `db.js` больше не должен содержать `localhost` и `root` как единственный вариант.
- `server.js` должен использовать `process.env.PORT`, иначе хостинг не сможет выбрать нужный порт.
- Если база пустая, сначала выполни `schema.sql`.

## Частые проблемы

**Ошибка: Access denied for user 'root'**  
→ Проверь пароль в файле `db.js`

**Ошибка: Unknown database 'servicedesk'**  
→ Еще раз выполни `schema.sql` через MySQL Workbench

**Порт 3000 занят**  
→ В `server.js` измени `3000` на `3001`
